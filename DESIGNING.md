# YARG Design Style
A game requires many pieces of a puzzle to fit together in order for it to feel, look, and to just be good. Because of this, a consistent philosophy and quality of work is required for it to fit together. This section contains information on YARG's design guidelines, so contributors know how to approach adding new content to YARG.

## YARG Design Goals
* **Robustness and a “Good Default”:** The goal of YARG is for it to be as fun as possible for new players. Because of this, it is important that we provide users with a good default experience—meaning, before changing any settings. All features of YARG should be as robust as possible, and not require external any tinkering to feel the best. In short, It should just work™️.

* **Customization, and “Modlessness”:** YARG’s end goal is to be as customizable as possible, so it suits the needs of all players, **without sacrificing competitive potential**. Modding of games usually happens because the game is missing a feature, or requires new content. YARG’s goal is for it to not require mods. All features that could be adjustable, should be. As for new content, the user should have the ability to do this without the need of external modification (i.e. new highways, etc.).

We have extended guidelines for contributing visuals in our Discord server, if you want to be a Art Contributor make sure to read the full rules so we can keep a good workflow.

## Design System
YARG's general artstyle and UI is meticulously planned on Figma before any development begins. Before contributing any art assets, be sure to reference the official Figma project to see if designs have already been made!

**[Click here to view the YARG Design System Figma project](https://www.figma.com/design/v2uJdQ03SKjIL18slAA1wD/YARG---Design-System--Original-?node-id=1-165&t=mluFHT9q9FwGY8U0-1)**

### Figma Project Outline
This Figma project is consistently updated with new designs for the in-game UI, websites, and other external applications we plan to build under the YARC umbrella.

- Overall Branding
  - Contains YARG's main branding elements, including colors, fonts, and logos, as well as special icons for things like instruments.
- Website
  - Contains plans for various YARG web projects, including a main webpage and leaderboards.
- Launcher
  - Cointains designs for the YARC Launcher's onboarding and marketplace sections.
- Game UI
  - Contains all of YARG's direct in-game user interface design. **If you plan on contributing any UI to YARG, please make sure you follow the designs in this file!**
- Sources
  - Non-comprehensive list of game source icons.
- YALCY
  - Contains UI prototypes for [YALCY](https://github.com/YARC-Official/YALCY), or Yet Another Lighting Control for YARG.
